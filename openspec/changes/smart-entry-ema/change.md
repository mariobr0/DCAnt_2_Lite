# Smart Entry EMA

## Зачем

Пользователю необходим механизм фильтрации входов (Smart Entry). Для начала реализуем простейший фильтр по скользящей средней (EMA): вход в новую сетку (Long) разрешен только в том случае, если текущая цена равна или ниже значения EMA за заданный период. Это позволяет боту избегать покупок на самом пике локального тренда и дожидаться отката.

## Требуемое поведение

1. **Новые настройки:**
   - В UI стратегии `DcaGridLiveTestStrategy` добавлен параметр `EMA Period` (по умолчанию 0, что означает "отключено").
   - В `DcaSettings` (или на уровне плагина, так как индикатор живет в Quantower) добавлена настройка периода EMA.
2. **Инициализация индикатора:**
   - В `OnRun` создается встроенный индикатор `Core.Indicators.BuiltIn.EMA` с заданным периодом и добавляется на график (с помощью `AddIndicator`).
3. **Логика входа:**
   - При старте нового цикла бот проверяет текущую цену `TestSymbol.Ask` (или `Last`) и значение индикатора `EMA`.
   - Если текущая цена `<= EMA` (или если фильтр отключен, т.е. период = 0), бот начинает цикл и выставляет ордера.
   - Если текущая цена `> EMA`, бот переходит в режим ожидания (постоянно проверяет цену на каждом тике или по таймеру, пока условие не будет выполнено). Как только цена опускается ниже EMA, цикл стартует.
4. **Ожидание после завершения:**
   - После завершения предыдущего цикла по TP или SL (и прохождения паузы `PauseAfterTpMs` / `PauseAfterSlMs`) бот не начинает новый цикл мгновенно, а снова переходит к проверке условия EMA.

## План реализации

- **[NEW/MODIFY] `DcaGridLiveTestStrategy.cs`:**
  - Добавить параметр `[InputParameter("EMA Period", 10)] public int EmaPeriod = 0;`
  - Добавить поле для индикатора `private Indicator _ema;` и инициализировать его в `OnRun`.
  - Добавить флаг состояния `private bool _isWaitingForEma = false;` (или просто проверять `_cycle == null` / завершен).
  - Убрать прямую отправку `StartNewCycleMessage` из таймеров паузы, вместо этого взводить флаг готовности.
  - В цикле `_throttleTimer` (или методе `OnUpdate` стратегии) проверять условие EMA, и если оно выполнено и мы готовы к старту — вызывать `ProcessStartNewCycle()`.
- **Проверка в симуляторе:**
  - Убедиться, что при цене выше EMA бот не открывает сделки.
  - Убедиться, что при пересечении ценой EMA вниз бот успешно открывает позицию.
