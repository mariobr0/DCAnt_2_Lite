# Асинхронный файловый логгер (Этап 9.5)

## Зачем

Quantower пишет логи стратегий в общие системные файлы вперемешку с тысячами других технических логов терминала. Для удобного мониторинга и отладки нам нужен отдельный, чистый текстовый лог-файл, который не зависит от UI терминала и гарантированно сохраняет всю хронологию торгов с миллисекундной точностью, не блокируя при этом рабочий поток торгового движка (`EngineLoop`).

## Требуемое поведение

- В `DCAnt2.Core` введен интерфейс `IBotLogger` (`Info`, `Error`), который используется ядром (`TradeCycle`, `EngineLoop`).
- В `DCAnt2.Plugin` реализован класс `QuantowerFileLogger`, пишущий логи через фоновый поток (`System.Threading.Channels`), чтобы не блокировать вызывающий поток (Non-blocking I/O).
- При старте стратегии создается уникальный текстовый файл (содержащий дату, время и GUID в имени).
- В начало файла с помощью рефлексии (Reflection) однократно записываются все текущие параметры стратегии.
- При остановке стратегии происходит принудительный сброс (Flush) оставшихся в памяти логов на диск, чтобы ничего не потерялось.
- Сообщения дублируются в стандартное окно Quantower через вызов встроенного `Strategy.Log()`.

## Не входит

- Использование сторонних тяжеловесных библиотек (NLog, Serilog и т.д.).
- Настройка ротации логов (удаление старых файлов по размеру/дате).
- Отправка логов по сети или в базы данных.
