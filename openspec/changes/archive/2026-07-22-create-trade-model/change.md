# Create Trade Model

## Зачем

Необходима "чистая" доменная торговая модель (`TradeCycle`), которая будет управлять жизненным циклом позиции. Она должна реагировать на рыночные события и исполнение ордеров, пересчитывать позицию и VWAP, генерировать торговые намерения (Intents) для выставления и перемещения ордеров, а также безопасно обрабатывать ошибки (режим `ExitOnly`).

## Требуемое поведение

- Создан агрегат `TradeCycle`, который принимает события (`OrderExecuted`, `OrderRejected`, `PriceTick` и т.д.).
- Позиция и средняя цена (VWAP) пересчитываются строго на основе подтвержденных `Executions` (без привязки к внешнему балансу `Account`).
- При исполнении каждого нового ордера сетки (DCA), бот формирует намерения (`CancelOrderIntent` для снятия старого и `PlaceOrderIntent` для выставления нового) для перемещения ордеров Take Profit и Stop Loss с учетом нового VWAP.
- При критической ошибке выставления DCA-ордера (например, недостаток средств `Insufficient Funds`), цикл переходит в безопасный режим `ExitOnly` (прекращает выставление новых DCA, но оставляет активными существующие TP/SL).
- Все исходящие команды бота формируются в виде строгих намерений (`Intents`), готовых для сохранения в паттерн Outbox.

## Не входит

- Интеграция с реальным API Quantower (слушатели событий, коннекторы).
- Инфраструктурный слой сохранения (SQLite Repository, Outbox worker).
- Визуальные интерфейсы.
