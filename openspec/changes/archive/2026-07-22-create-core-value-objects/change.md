# Create Core Value Objects

## Зачем

Необходимо заложить строгий фундамент доменной модели, чтобы предотвратить ошибки смешивания идентификаторов (например, передача ID ордера вместо ID цикла) и избежать потери точности в финансовых расчетах. Читаемые строковые ID с префиксами и датой кардинально упростят отладку по логам.

## Требуемое поведение

- Добавлены строго типизированные идентификаторы (`StrategyInstanceId`, `TradeCycleId`, `InternalOrderId`, `EffectId`), оборачивающие строку.
- Идентификаторы используют префикс, дату и случайный хвост (например, `ord_260722_a1b2c3d4e5f6`), гарантированно укладываясь в лимиты длины `ClientRequestId`.
- Разные типы ID не могут быть приравнены или переданы вместо друг друга (ошибка компиляции).
- Финансовые типы (`Money`, `Quantity`, `Price`) являются `readonly record struct` и используют строго `decimal`.
- Класс `InstrumentRules` содержит параметры инструмента (`TickSize`, `QuantityStep`, `MinNotional`) и предоставляет методы: `RoundPrice` (математическое) и `RoundQuantityDown` (строго вниз).

## Не входит

- Поддержка мультивалютности внутри одного инстанса стратегии.
- Реализация автоинкрементных счетчиков для ID.
- Настройка маппинга этих типов для базы данных (Dapper TypeHandlers).
- Подключение к Quantower API.
