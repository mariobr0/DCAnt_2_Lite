# Практический тест-драйв (Этап 8.5)

## Зачем

Прежде чем усложнять архитектуру и переходить к полному DCA-циклу (Этап 9), необходимо убедиться, что базовая интеграция компонентов (`EngineLoop`, `SQLite`, `QuantowerAdapter`) корректно работает в реальной среде терминала Quantower. Нам нужен простейший плагин (стратегия), который запустит созданные модули и проведет одну реальную сделку, чтобы выявить возможные проблемы на стыке с API биржи.

## Требуемое поведение

- Создана стратегия `SingleTradeLiveTestStrategy` внутри `DCAnt2.Plugin`.
- Стратегия инициализирует `SqliteTradingStateStore` (создавая файл БД) и запускает `EngineLoop` в фоновом потоке.
- Стратегия инициализирует `QuantowerAdapter` и передает ему управление ордерами.
- По кнопке "Start" бот выставляет один лимитный ордер (вход в позицию) и фиксирует намерение в Outbox.
- После исполнения входа бот корректно перехватывает событие (Execution) и выставляет Take Profit.
- При перезапуске плагина (Stop -> Start) бот корректно считывает историю из Quantower API и не дублирует ордера (Reconciliation).

## Не входит

- Полноценный пользовательский интерфейс (UI) стратегии (настройки задаются в коде жестко или простейшими параметрами).
- Работа с DCA-сеткой.
- Автоматическое тестирование в CI/CD (этот тест выполняется пользователем вручную в терминале).
