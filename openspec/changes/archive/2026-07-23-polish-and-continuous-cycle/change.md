# Polish and Continuous Cycle

## Зачем

Текущая реализация стратегии завершает работу сразу после закрытия одной сетки по Take Profit или срабатывания ошибки. Для DCA-бота критически важно работать в непрерывном режиме (всегда включен по умолчанию). После закрытия по Take Profit бот должен подождать заданную паузу и начать новый цикл. После Stop Loss пауза будет отдельной (настраиваемой независимо).
Также необходимо вывести параметры торговой логики (шаг, тейк-профит) в графический интерфейс Quantower, чтобы их можно было менять без перекомпиляции кода, и устранить обнаруженную гонку потоков при асинхронном получении отказов от биржи.

## Требуемое поведение

- **Непрерывный цикл**: Бот всегда работает в непрерывном режиме. Если цикл успешно закрыт по TP (позиция равна 0, ордеров нет), `EngineLoop` отменяет оставшиеся лимитки и через заданную `Паузу после Take Profit` (в миллисекундах) автоматически инициирует новый `TradeCycle`.
- **Пауза после Stop Loss**: Если цикл закрыт по SL, бот использует отдельную настройку `Паузы после Stop Loss` перед рестартом.
- **Настройки в UI**: Доступные через `[InputParameter]` настройки стратегии Quantower: `Пауза после TP`, `Пауза после SL`, `Шаг сетки (%)`, `Take Profit (%)`, `Множитель объема`, `Макс. уровней сетки`, `Активное окно сетки`.
- **Устранение гонки потоков**: Диспетчеризация первого (рыночного) ордера не должна конфликтовать с фоновым потоком `EngineLoop`.
- **Очистка стейта**: Перед стартом нового цикла бот генерирует новый `TradeCycleId`, полностью сбрасывая Outbox и старые привязки.

## Не входит

- Рисование уровней на графике (будет добавлено позже, чтобы не перегружать задачу).
- Умные индикаторы входа (EMA, MFI) — остаются на Этап 10.
- Сложный менеджмент риск-лимитов для непрерывного цикла (берем доступный баланс).
