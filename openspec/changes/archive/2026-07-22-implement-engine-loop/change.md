# Реализация EngineLoop

## Зачем
Доменное ядро (`TradeCycle`) не является потокобезопасным (не использует lock). Чтобы гарантировать отсутствие состояний гонки (race conditions) при одновременном поступлении тиков и ответов биржи, необходим строгий однопоточный обработчик (Actor Model), который выстраивает все события в единую очередь.

## Требуемое поведение
- Все внешние триггеры оформляются в виде объектов `EngineMessage` (рыночная цена, исполнение ордера, ответ биржи).
- Очередь событий реализована через `System.Threading.Channels.Channel<EngineMessage>`.
- Ровно один фоновый поток (Consumer) достает сообщения по одному и передает их в `TradeCycle`.
- Новая рыночная цена попадает в канал через паттерн Polling: внутренний таймер (по умолчанию 250мс) запрашивает цену и генерирует `MarketQuoteMessage` во избежание переполнения (Quote Storms).
- Системные ошибки и `Exception` внутри цикла (Panic) приводят к жесткой остановке `EngineLoop` (отключение бота) для защиты состояния. Ранее выставленные биржевые ордера остаются активными.
- Торговые ошибки (`OrderRejected`) маршрутизируются в `TradeCycle` и переводят его в режим `ExitOnly` (цикл `EngineLoop` продолжает работать).
- Реализован механизм контролируемой остановки цикла (Controlled Shutdown).

## Не входит
- Сохранение состояния в базу данных (SQLite/Outbox — это Этап 6).
- Обработка кнопок UI.
- Реальная привязка к Quantower API (пока разрабатывается только инфраструктура движка).
