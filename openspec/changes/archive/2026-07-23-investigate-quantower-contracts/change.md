# Исследование контрактов Quantower API (Contract Tests)

## Зачем

Прежде чем реализовывать торговую логику для выставления реальных ордеров, мы должны точно знать, как ведет себя выбранный коннектор в Quantower. Документация может отличаться от реальности для конкретной биржи. Нам нужно проверить защиту от дубликатов, таймауты и возможность надежного поиска наших ордеров.

## Требуемое поведение

- Проверить, обеспечивает ли `ClientRequestId` (или `Comment`) фактическую защиту от повторной отправки (идемпотентность) на выбранном коннекторе.
- Проверить возможность поиска ордера по `ClientRequestId` (или `Comment`) после перезапуска терминала.
- Проверить срок и область уникальности `ClientRequestId` (или `Comment`).
- Проверить фактическое поведение rate limits и timeout (как долго ждать ответа).
- Определить, какие read-only запросы (например, запрос списка ордеров) допускают автоматический retry.

## Не входит

- Реализация торговой логики отправки ордеров из `Outbox`.
- Написание полного DCA-цикла (всё это перенесено на следующие этапы).
- Реализация UI бота.
