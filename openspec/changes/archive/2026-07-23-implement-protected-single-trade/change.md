# Минимальная защищенная сделка (Этап 8)

## Зачем

Требуется объединить уже созданные модули (Core, SQLite Outbox, Quantower Adapter) в единый рабочий процесс для проведения одной полной торговой сделки. Это базовая проверка того, что бот способен войти в рынок, дождаться исполнения ордера, выставить Take Profit / Stop Loss и безопасно завершить сделку, обрабатывая возможные сбои на пути (отказы, частичные исполнения, разрывы связи).

## Требуемое поведение

- Завершенный торговый цикл: Market Entry → Execution → Position Active → TP/SL → Exit Execution → TradeCycle Completed.
- Реализована идемпотентная логика Reconciliation (докачка истории) при старте с использованием `ExecutionId` и `ClientRequestId`.
- Адаптер Quantower надежно транслирует намерения из `Outbox` в команды API.
- Адаптер Quantower слушает события биржи (Executions, Order Updates) и передает их в виде сообщений в `EngineLoop`.
- Корректная обработка: partial fills, entry rejection, entry timeout, late executions и duplicate events.
- Обеспечена безопасная перестановка ордеров (с подтверждением отмены перед новой установкой).

## Не входит

- Полноценная логика DCA-сетки (размещение уровней, масштабирование шага и объема).
- Short-направление.
- Динамические индикаторы и сложные виды стопов.
- Пользовательский интерфейс плагина (UI).
