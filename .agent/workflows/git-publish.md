# Git Publish

Опубликуй текущие изменения в GitHub.

Выполни по порядку:

1. Покажи краткий `git status`.
2. Проверь, что в изменениях нет:
   - паролей и токенов;
   - `.env`;
   - логов;
   - рабочих баз данных;
   - папок `bin` и `obj`.
3. Если найден solution, выполни Release-сборку и тесты.
4. Если сборка или тесты не прошли, остановись и объясни ошибку.
5. Составь короткое понятное название коммита на английском языке.
6. Добавь проверенные изменения в Git.
7. Создай один коммит.
8. Отправь текущую ветку в `origin`.
9. Покажи:
   - название коммита;
   - hash коммита;
   - имя ветки;
   - результат отправки.

Не изменяй код во время этого workflow.

Не используй:

- force push;
- reset --hard;
- удаление веток;
- изменение истории коммитов.

Если изменений нет, ничего не делай.
Если `origin` не настроен, остановись и объясни проблему.
