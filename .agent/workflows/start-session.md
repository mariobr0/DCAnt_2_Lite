# Start Development Session

Подготовься к продолжению разработки DCAnt 2

Ничего не изменяй в проекте и не пиши код.

## 1. Изучи постоянный контекст

Прочитай:

- `docs/PROJECT.md`;
- `docs/ARCHITECTURE.md`;
- `docs/ROADMAP.md`;
- `openspec/config.yaml`;
- `docs/changelog_ru.md`;
- `docs/quantower_contract_analysis.md`;
- `docs/references.md`;
- `docs/Не пересказывай эти документы целиком`.

## 2. Найди текущую задачу

Проверь каталог:

`openspec/changes/`

Найди активный change, не находящийся в архиве.

Если активный change существует:

- прочитай его `change.md`;
- определи цель изменения;
- проверь раздел `Требуемое поведение`;
- проверь выполненные и невыполненные задачи;
- проверь раздел `Результат проверки`.

Если активных changes несколько, перечисли их и укажи, какой из них выглядит текущим. 

Ничего не выбирай и не изменяй автоматически.

Если активного change нет, сообщи об этом прямо.

## 3. Проверь состояние Git

Выполни:

- `git status --short`;
- `git branch --show-current`;
- `git log -1 --oneline`.

Определи:

- текущую ветку;
- последний commit;
- наличие незакоммиченных изменений.

Не создавай commit и не выполняй push.

## 4. Проверь структуру проекта

Кратко проверь:

- наличие solution;
- наличие production-проектов;
- наличие тестовых проектов;
- существование файлов, упомянутых в текущем `change.md`.

Не выполняй полную сборку и тесты без отдельного запроса.

## 5. Дай краткий отчет

Ответь простым русским языком в следующем формате:

### Где мы сейчас

Укажи текущий этап разработки и активный change.

### Что уже сделано

Перечисли только фактически завершенные части текущей задачи.

### Что осталось

Перечисли невыполненные пункты `change.md`.

### Состояние Git

Укажи:

- текущую ветку;
- последний commit;
- наличие незакоммиченных изменений.

### Что требует внимания

Укажи противоречия, незавершенные проверки или несколько активных changes.
Если проблем нет, напиши:
`Проблем нет. Контекст проекта согласован.`

### Следующий шаг

Назови один следующий шаг, который прямо следует из текущего `change.md` или `ROADMAP.md`.

Не начинай выполнять этот шаг автоматически.

## Правила

- Не изменяй файлы.
- Не ставь галочки в `change.md`.
- Не пиши production-код.
- Не создавай новый OpenSpec change.
- Не выполняй commit или push.
- Не запускай длительные проверки.
- Не читай архивные changes без необходимости.
- Не загружай несвязанные документы.
- Не придумывай прогресс, которого нет в файлах или Git.
- Если информация противоречива, скажи об этом прямо.
- Объясняй результат естественно и без лишней технической детализации.
