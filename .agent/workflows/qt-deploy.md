# Workflow: Деплой стратегии в Quantower

**Когда использовать:**
Каждый раз, когда вносятся изменения в плагин или логику, и нужно проверить работу бота в Quantower.

**Что делает этот workflow:**
1. Вызывает скрипт `deploy.ps1`, который выполняет сборку `dotnet build -c Release`.
2. Копирует скомпилированные DLL-файлы (`DCAnt2.Plugin.dll`, `DCAnt2.Quantower.dll`, `DCAnt2.Core.dll`, `DCAnt2.Infrastructure.dll`) в папку `C:\Users\dzam\Desktop\APPs\Quantower\Settings\Scripts\Strategies`.
3. Напоминает пользователю, что тестовая стратегия автоматически получает **уникальный суффикс со временем сборки** (например, `[12:05:30]`), чтобы можно было отличить свежую версию от старой закэшированной версии терминала.

**Шаги для агента:**
1. Выполни команду `.\deploy.ps1` в PowerShell.
2. Проверь отсутствие ошибок компиляции.
3. Сообщи пользователю точное время сборки (суффикс), которое он должен искать в списке стратегий Quantower.
