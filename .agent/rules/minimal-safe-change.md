# Минимальное безопасное изменение

Перед созданием нового элемента проверь:

1. Нужен ли он для текущего change.md?
2. Есть ли подходящее решение уже в проекте?
3. Можно ли изменить существующий компонент?
4. Можно ли обойтись без нового проекта, интерфейса или зависимости?
5. Что произойдет, если отложить его создание?

Не сокращай:

- order ownership;
- защиту от duplicate orders;
- подтверждение отмены перед заменой ордера;
- обработку partial fills;
- execution deduplication;
- Outbox;
- recovery;
- reconciliation;
- валидацию;
- важные логи;
- критические тесты.

Краткость не имеет приоритета над корректностью торгового поведения.
