# Контроль сложности проекта

Не увеличивай сложность DCAnt_2_Lite без доказанной необходимости.

Без отдельного согласования пользователя запрещено:

- создавать новый проект solution;
- добавлять архитектурный слой;
- создавать новый архитектурный документ;
- создавать ADR;
- вводить новую абстракцию «на будущее»;
- добавлять функциональность, отсутствующую в утвержденном change;
- переносить отложенное улучшение в текущую реализацию.

Перед предложением новой сущности кратко ответь:

1. Какую текущую проблему она решает?
2. Почему существующая структура не решает эту проблему?
3. Что произойдет, если отложить ее?
4. Является ли она обязательной на текущем этапе?

Не создавай новый ADR, если не меняется уже принятое долгосрочное архитектурное решение.

Не создавай отдельный OpenSpec change для:

- исправления опечатки;
- форматирования;
- изменения комментария;
- локального рефакторинга без изменения поведения;
- обновления документа без изменения требований;
- теста уже утвержденного поведения.
